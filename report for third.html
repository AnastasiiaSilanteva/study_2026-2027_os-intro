Введение

Цель работы: изучить базовый синтаксис Markdown и освоить навыки оформления технических отчётов с его использованием. В ходе работы будет подготовлен отчёт по предыдущей лабораторной работе (№2) в формате Markdown и произведена его компиляция в PDF и DOCX.

1. Теоретические сведения о Markdown

Markdown — облегчённый язык разметки, созданный для удобного написания текстов с простым форматированием и последующего преобразования в HTML или другие форматы. Ниже приведены основные элементы синтаксиса, использованные в данном отчёте.

1.1 Заголовки

Заголовок 1 уровня

Заголовок 2 уровня

Заголовок 3 уровня

Заголовок 4 уровня

1.2 Форматирование текста

Жирный текст — жирный

Курсив — курсив

Жирный курсив — жирный курсив

Зачёркнутый — зачёркнутый

1.3 Списки

Ненумерованный:

markdown

  • Элемент 1
  • Элемент 2
    • Вложенный элемент (с отступом 2 или 4 пробела)

Нумерованный:

markdown

  1. Первый пункт
  2. Второй пункт
    1. Подпункт 2.1

1.4 Цитаты

Цитаты оформляются символом > в начале строки.

1.5 Блоки кода

Блоки кода выделяются тройными обратными кавычками с указанием языка:

bash git add . git commit -m "Initial commit"

1.6 Ссылки

markdown текст ссылки

1.7 Изображения с подписями и перекрёстными ссылками

markdown { #fig:метка } Ссылка на рисунок: [-@fig:метка].

История Последовательность коммитов
