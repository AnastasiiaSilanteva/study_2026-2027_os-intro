Введение

Подготовка к выполнению лабораторной работы

1. Подготовка macOS

Все действия выполнялись в приложении Терминал (Terminal).

1.1 Установка Fedora

Я заранее установила Fedora-Sway-Live-x86_64-43-1.6.iso

2. Выполнение лабораторной работы

2.1 Установка Homebrew

Я установила homebrew — это менеджер пакетов для macOS, через который мы установим QEMU.

После завершения Homebrew выводит сообщение ==> Next steps:. В нём есть две команды, которые нужно скопировать и выполнить. Обязательно выполняем их и проверяем установку.

Далее я устанавливаю qemu.

Создаю папку virtlab в вашей домашней папке, чтобы хранить всё: образ диска, ISO-файл, скрипты запуска. Я также перемещаю скачанный ISO-образ Fedora в эту папку.

Далее я создаю образ виртуального диска.

И наконец, запускаем QEMU в режиме полной эмуляции системы для архитектуры x86_64.

qemu-system-x86_64 — запускаем эмулятор для 64-битной x86 системы.

machine type=q35,accel=tcg — используем современный чипсет Q35 и программную эмуляцию (tcg), так как KVM на Mac недоступен.

cpu max - эмулируем максимально возможный процессор.

smp 4 — выделяем 4 виртуальных ядра.

m 2048 — 2 ГБ оперативной памяти.

drive file=... — подключаем наш виртуальный жёсткий диск.

cdrom ... — подключаем ISO-образ Fedora как CD-ROM.

boot menu=on — при старте показывать меню выбора загрузочного устройства (чтобы можно было выбрать CD).

vga virtio — тип видеокарты.

display default,show-cursor=on — стандартное отображение окна, показывать курсор.

device virtio-balloon — драйвер для управления памятью (гость может отдавать память хосту).

chardev ... -device ... — параметры для буфера обмена (чтобы работало копирование текста между хостом и гостем).

2.2 Установка Fedora на диск

Когда у меня загрузился графический интерфейс Fedora (Sway), я увидила приглашение и открыла терминал(ввела комаду liveinst).

Запустился графический установщик Anaconda и я завершила настройки.

3. Настройка гостевой системы (внутри виртуалки)

3.1 Установка драйверов для QEMU

Запускаем tmux (терминальный мультиплексор, чтобы не потерять сессию, если что). И установливает небходимые пакеты.

И перезагружаем виртуальную машину с помощью reboot.

3.2 Обновление системы и установка дополнительного ПО

Далее я отключаю SELinux и настраиваем переключение клавиатуры(с помощью caps lock)

Докачиваем ПО для создания отчета(pandoc).

4. Выполнение домашнего задания

После установки и настройки системы выполнила команду dmesg для анализа загрузки:

И тем самым находи всю необходимую информацию.

Версия ядра: dmesg | grep "Linux version"

Частота процессора: dmesg | grep -i "detected.*MHz"

Модель процессора: dmesg | grep "CPU0"

Объём доступной памяти: dmesg | grep -i "Memory.*available"

Тип гипервизора: dmesg | grep -i "Hypervisor detected"

Тип ФС корневого раздела: dmesg | grep "rootfs" или посмотрите mount | grep " / ".

Последовательность монтирования: dmesg | grep -i "mount" или journalctl -b | grep -i "mount".

5. Ответы на контрольные вопросы

Учётная запись пользователя содержит: имя пользователя (login), UID (идентификатор пользователя), GID (основной группы), домашний каталог, командную оболочку (shell), пароль (обычно в зашифрованном виде в /etc/shadow), комментарий (полное имя и т.д.). Команды:

Справка: man <команда> или <команда> --help. Перемещение: cd <путь>. Просмотр содержимого каталога: ls [опции] <путь>. Определение объёма каталога: du -sh <каталог>. Создание каталога: mkdir <имя>. Удаление каталога: rmdir <пустой_каталог> или rm -rf <каталог>. Создание файла: touch <имя> или > <имя>. Удаление файла: rm <файл>. Задание прав: chmod <режим> <файл>. Просмотр истории команд: history. Файловая система — способ организации и хранения данных на носителе. Примеры: ext4 (типовая для Linux), XFS (высокопроизводительная), Btrfs (с поддержкой снапшотов), FAT32 (совместимая с Windows), NTFS (Windows). Посмотреть примонтированные ФС: mount или findmnt, или df -hT. Удалить зависший процесс: найти его PID через ps aux | grep <имя_процесса> или top, затем kill -9 .
